# Windows 英语听写填空应用设计规格

日期：2026-07-22  
状态：已完成交互式设计确认，等待书面规格复核  
工作名称：听写填空

## 1. 产品目标

开发一个只供家庭内部使用的 Windows 11 x64 离线英语学习应用。应用播放完整英文句子的本地合成音频，同时展示挖去一个单词或短语后的句子。用户通过听音频填写所有空位，应用即时判断答案。

首版的核心价值是稳定、专注的听写体验：无需账号、无需服务器、无需联网，安装后即可练习，并能根据首次答题结果调整无尽模式的难度。

## 2. 目标平台与交付形态

- 目标操作系统：Windows 11 x64。
- 开发环境：先在 macOS 上完成跨平台代码、自动化测试和界面开发。
- 最终构建与验收：在 Windows 11 x64 真机完成。
- 运行环境：目标电脑无需预装 Python。
- 打包方式：PyInstaller 生成目录式应用，再由 Inno Setup 生成 `ListeningClozeSetup.exe` 安装程序。目录式打包避免大型 TTS 模型在每次启动时解压。
- 首版不发布到 Microsoft Store，不公开分发，不盈利。

## 3. 技术栈

- Python 3.12
- PySide6 + Qt Quick/QML
- Qt Multimedia
- Supertonic 3 Python SDK + ONNX Runtime CPU 推理
- SQLite，使用 Python 标准库 `sqlite3`
- spaCy + wordfreq + 自定义英文短语规则
- pytest + pytest-qt + pytest-xdist
- uv 管理依赖与开发命令
- PyInstaller + Inno Setup 打包 Windows 安装程序

Supertonic 模型、分词资源和美式女声文件随安装包提供。运行时必须使用本地资产路径，并禁用自动下载。

## 4. 视觉设计

已批准的视觉方向为现代 Windows Fluent 2 风格：

- 白色或浅灰背景、克制的蓝色主色、绿色成功状态和红色错误状态。
- 10–14px 圆角、轻量阴影、清晰的 Segoe UI 字体层级。
- 核心练习卡占据主要空间，不使用大型侧边栏或密集仪表盘。
- QML 使用自定义控件风格，不依赖 Qt 默认控件皮肤。
- 1440×900、100% 缩放时高度还原已批准原型。
- Windows 125% 与 150% DPI 下不得出现截断、重叠或模糊。
- Windows 原生标题栏允许存在系统级细微差异。

视觉组件包括：

- 应用标题与离线状态。
- 难度、内容分类、题目进度标签。
- 播放按钮、真实音频波形、重听按钮和 0.8×/1.0×/1.2× 语速选择。
- 单词或短语输入框。每个规范答案单词对应一个可见空位。
- 正确、错误、加载和不可用状态。
- 每轮题目进度与摘要。
- 右上角蓝色声波精灵。

## 5. 页面范围

### 5.1 首页

- 进入定量练习或无尽模式。
- 显示已练习题数、待掌握题数和最近一次练习。
- 有未完成练习时显示“继续上次练习”。

### 5.2 定量练习配置

- 内容分类：日常口语、考试英语、影视表达、新闻/播客或全部分类。
- 难度：简单、中等、困难。
- 题量：10、20、30。

### 5.3 练习页

- 播放、重听、语速切换。
- 展示挖空句子并接受键盘输入。
- Enter 提交答案；完成后 Enter 进入下一题。
- 答错后允许无限重听和修改，也可主动查看答案。
- 显示声波精灵动画和随机反馈文案。

### 5.4 定量练习结算页

- 正确数、错误数、查看答案数、重听次数。
- 支持立即复习本轮首次答错或查看答案的题目。

### 5.5 无尽模式结束页

- 完成题数、正确率、最高难度、结束时难度和最长连续答对。

### 5.6 设置页

- 默认语速、音量、动画开关。
- 音频缓存数量与占用空间。
- 重置学习记录。

### 5.7 启动修复页

- 仅在题库、TTS 模型或女声资产缺失或损坏时出现。
- 明确列出问题文件及修复说明，禁止进入无内容的空白主界面。

## 6. 分层架构

```text
PySide6 / QML 界面
        ↓
练习控制器与页面状态
        ↓
题目生成、答案判定、选题调度
        ↓
题库数据库 / 学习记录 / TTS / 音频缓存
```

建议目录职责：

- `src/listening_cloze/ui/`：QML 页面、自定义控件、图标和动画资源。
- `src/listening_cloze/application/`：QML 桥接对象、页面状态和练习控制器。
- `src/listening_cloze/domain/`：题目、答案判定、练习会话、难度状态机和选题策略。
- `src/listening_cloze/infrastructure/`：SQLite、TTS、缓存、音频播放适配器和配置。
- `tools/content_pipeline/`：网页收集、清洗、去重、候选答案分析和数据库构建。
- `tests/`：领域、基础设施、流水线、QML 和集成测试。

QML 不直接访问 SQLite、文件系统或 Supertonic。QML 只调用公开的 Python 控制器属性、信号和槽，业务规则保持在可独立测试的 Python 模块中。

## 7. 数据设计

### 7.1 只读题库 `content.db`

每个原句包含：

- 稳定 ID
- 完整英文句子
- 内容分类
- 来源 URL
- 规范化文本哈希
- 三个难度版本的关联关系

每个题目版本包含：

- 原句 ID
- 难度：`easy`、`medium`、`hard`
- 答案起止位置
- 规范答案
- 规范答案单词数
- 等价答案列表
- 自动难度分值与生成依据

### 7.2 可写学习库 `user.db`

记录：

- 每题首次结果
- 总尝试次数
- 重听次数
- 是否查看答案
- 最近练习时间
- 正确与错误累计
- 未完成会话及题目顺序
- 无尽模式当前难度、连续正确数和连续错误数
- 用户设置
- 数据库 schema 版本

`content.db` 与 `user.db` 分离，应用或题库升级不得覆盖学习记录。

## 8. 题库构建

### 8.1 规模与分布

- 300 个原句。
- 日常口语、考试英语、影视表达、新闻/播客原则上各 75 个原句。
- 每个原句生成简单、中等、困难三个不同版本。
- 最终共 900 道题。

### 8.2 收集流程

构建工具从配置的公开网页、RSS、字幕或文本页面提取候选英文句子，并保存来源 URL。题库仅用于家庭内部使用；如未来公开分发，必须重新执行版权审核并替换没有明确再分发许可的内容。

清洗规则包括：

- 移除 HTML、不可见字符和异常 Unicode。
- 统一引号和空白。
- 按规范化文本去重。
- 排除过短、过长、残缺和主要由数字组成的句子。
- 排除明显不适合学习的敏感或低质量内容。

### 8.3 自动挖空

使用 spaCy 词性与依存关系、wordfreq 词频以及自定义短语动词和固定搭配规则，提取单词及多词短语候选。

- 简单：高频实词或上下文容易预测的基础短语。
- 中等：中频词、常用搭配或 2–3 词短语。
- 困难：低频词、习语、复杂搭配或较长短语。

三个版本必须使用不同答案。无法稳定生成三个递增难度版本的句子淘汰，并继续收集替代句，直到满足 300 个原句。

### 8.4 自动质量门禁

- 答案必须完整存在于原句中。
- 填回规范答案后必须与原句完全一致。
- 可见空位数量严格等于规范答案单词数。
- 不挖纯标点、孤立冠词、数字或无学习价值的专有名称。
- 避免答案在同一句中重复导致歧义。
- 三个版本难度分值必须严格递增。
- 生成常见缩写等价答案，如 `do not`/`don't`、`cannot`/`can't`。
- 输出题库质量报告：来源分布、分类分布、难度分布、短语长度、重复率和失败原因。

## 9. 答案输入与判定

### 9.1 空位与短语

规范答案中的每个空格分隔单词对应一个可见输入框。例如 `take part in` 显示三个空位。

输入框允许粘贴或输入包含空格的文本。提交时将所有非空输入框按顺序合并，因此即使等价答案与规范答案的单词数不同，也可判定：

- 规范答案为 `do not` 时显示两个空位；第一个空位输入 `don't`、第二个留空也可提交并判为等价。
- 规范答案为 `can't` 时显示一个空位；该输入框输入 `can not` 也可判为等价。

### 9.2 规范化规则

- 忽略大小写。
- 忽略首尾空格。
- 将连续空格折叠为一个空格。
- 将弯引号和直引号规范为同一种撇号。
- 除题库明确配置的等价答案外，不接受拼写错误或近似答案。

### 9.3 首次提交规则

- 第一次提交正确：本题记为正确。
- 第一次提交错误：立即记为错误，后续修改正确不改变本题统计结果。
- 主动查看答案：本题记为错误。
- 答错后可无限重听和修改，直到答对或查看答案。

## 10. 练习模式

### 10.1 定量练习

用户选择分类、难度和 10/20/30 题。开始时确定整轮题目顺序并保存会话。

### 10.2 无尽模式

- 每次新建无尽模式会话默认从简单开始。
- 连续答对 5 题：简单升级到中等，中等升级到困难。
- 连续答错 5 题：困难降到中等，中等降到简单。
- 答对时连续错误计数清零；答错时连续正确计数清零。
- 未达到 5 题时难度不变。
- 已在困难难度连续答对 5 题时保持困难并重置连续正确计数。
- 已在简单难度连续答错 5 题时保持简单并重置连续错误计数。
- 恢复未完成会话时保留恢复前的难度和连续计数；只有新会话默认简单。
- 用户可以随时结束并查看统计。

难度变化后，尚未执行的后续预取题目作废，并根据新难度重建队列。已经生成的音频缓存继续保留。

## 11. 选题策略

在符合当前分类和难度的候选集中：

1. 优先未做题。
2. 其次选择历史首次答错或查看过答案的题。
3. 已掌握题降低权重，但不永久排除。
4. 同一原句的不同难度版本不得在短期内连续出现。
5. 随机选择必须支持注入随机种子，以便测试可重复。

定量练习的整轮题目开始时固定。无尽模式使用滚动队列，始终维护当前题和后两题。

## 12. TTS 与音频缓存

### 12.1 模型配置

- Supertonic 3。
- 固定美式女声，最终女声从至少 20 个代表性句子的盲听样音中选择。
- CPU ONNX 推理，不要求独立显卡。
- 模型在应用启动后由后台 TTS 工作线程加载一次。

### 12.2 预生成队列

- 当前题：最高优先级。
- 下一题：第二优先级。
- 下下题：第三优先级。
- 用户进入下一题后，立即补入新的下下题。
- 用户快速切题时重新排列未执行任务。
- 同一原句的并发请求合并为一个生成任务。
- 相同原句的三个难度版本共享音频。

### 12.3 缓存

- 缓存键：规范化完整句子、TTS 模型版本和女声版本的稳定哈希。
- 语速不进入缓存键；播放器使用 `playbackRate` 处理 0.8×、1.0×、1.2×。
- 音频先写临时文件，完成校验后原子替换为正式 WAV。
- 缓存损坏或模型版本变化时自动重新生成。
- 300 个原句规模下不主动清理音频缓存。

### 12.4 失败处理

- 当前题生成失败时自动重试一次。
- 第二次失败后显示原因、“重新生成”和“跳过本题”。
- 磁盘空间不足时暂停后台预生成并提示用户。
- TTS 工作不得阻塞 QML 主线程。

## 13. 声波精灵

声波精灵位于练习卡右上角，仅承担情绪反馈，不遮挡主要操作。

### 13.1 待机

- 轻微呼吸、左右摆动或声波柱起伏。
- 动画关闭时保持静态。

### 13.2 答对动画池

- 原地弹跳并挥手。
- 拍手。
- 快速旋转一圈。
- 声波柱向上伸展。
- 抛出少量彩纸。
- 连续答对 5 题时播放专属升级动画。

答对文案池至少包含：

- 太棒了，听得很准！
- 完全正确！
- 这句听懂了！
- 漂亮，继续保持！
- 你的耳朵很敏锐！

### 13.3 答错动画池

- 低头下垂。
- 轻轻摇头。
- 声波柱缩短。
- 蹲下后重新站起。
- 小幅左右晃动。
- 连续答错 5 题时播放温和的难度调整动画。

答错文案池至少包含：

- 别灰心，再听一次！
- 差一点，再试试！
- 放慢语速听听看。
- 注意句子的连接发音。
- 没关系，再挑战一次！

升级与降级使用独立文案池。相同文案和相同动画不得连续重复。动画不得阻塞输入、播放或进入下一题。

## 14. 持久化与恢复

- 每次首次判分后立即保存。
- 保存定量练习的完整顺序和当前位置。
- 保存无尽模式的当前难度、连续计数和统计。
- 应用意外关闭后可继续未完成会话。
- `user.db` schema 升级前创建本地备份，只保留最近三个备份。
- 用户数据默认位于 `%LOCALAPPDATA%\ListeningCloze\`。
- 卸载时提示是否保留学习记录和音频缓存。
- 不上传学习记录、输入内容、日志或任何遥测数据。

## 15. 异常状态

- 音频生成中：播放按钮显示加载动画，输入仍可操作。
- 音频不可用：显示原因及重新生成按钮。
- 题库或模型缺失：显示启动修复页。
- 数据库迁移失败：保留原库与备份，禁止静默覆盖。
- 本地日志仅记录模块、错误类型和堆栈，不记录用户答案文本。

## 16. 测试策略

开发遵循测试驱动流程：先写失败测试，再实现最小代码，通过后重构。

### 16.1 领域测试

- 大小写、空白、撇号和等价答案。
- 多词短语、多输入框和不同单词数的等价答案。
- 首次提交错误后修改正确仍计为错误。
- 无尽模式连续 5 题升降级及边界行为。
- 选题权重、短期重复限制和可重复随机。

### 16.2 TTS 与缓存测试

- 当前题和后两题优先级。
- 快速切题重排。
- 重复任务合并。
- 缓存命中、损坏重建、模型版本失效和原子写入。
- TTS 失败重试与错误状态。

### 16.3 题库流水线测试

- 最终精确包含 300 个原句和 900 道题。
- 每句三个难度且答案不同、难度严格递增。
- 空位数量与规范答案单词数一致。
- 重建句子与原句一致。
- 分类、来源、重复率和淘汰原因报告完整。

### 16.4 持久化测试

- 首次判分立即保存。
- 会话恢复。
- schema 迁移、备份和失败回滚。

### 16.5 QML 与视觉测试

- 键盘输入、Tab 切换、Enter 提交和进入下一题。
- 播放、重听、查看答案、加载及错误状态。
- 精灵动画和文案不连续重复。
- 1440×900 的批准状态截图对比。
- Windows 100%、125%、150% DPI 真机检查。

自动化测试默认使用 `pytest -n auto` 并行运行。若发现共享资源冲突，再针对相关测试组回退为串行定位。

## 17. 最终验收标准

- 在未安装 Python 的 Windows 11 x64 电脑上可安装、启动和卸载。
- 断网环境下完成定量练习与无尽模式。
- 应用启动后 5 秒内界面可操作。
- 首题 TTS 生成时界面无明显卡顿。
- 后两题预取成功后，切题 200ms 内可开始播放。
- 答案、等价答案、首次结果和无尽模式升降级符合本规格。
- 崩溃或强制关闭后，已经提交的学习记录不丢失。
- 视觉效果符合批准原型，125% 与 150% DPI 无布局问题。
- 精灵动画丰富但不干扰练习。

## 18. 首版不包含

- 登录、账号和云同步。
- 联网题库市场或运行时下载。
- 用户导入题库。
- 多用户档案。
- 排行榜、社交、成就系统。
- 在线 AI 讲解。
- 自动更新程序。
- Microsoft Store 发布。
- iOS 或其他移动端构建。
